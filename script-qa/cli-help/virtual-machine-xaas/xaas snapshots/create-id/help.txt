Restore a snapshot of a Virtual Machine with the current tenant which is logged in. </br>
A Snapshot is ready for restore when it's in available state.

#### Notes
- You can verify the state of snapshot using the snapshot list command.
- Use machine-types list to see all machine types available.

#### Rules
- To restore a snapshot  you have to inform the new virtual machine information.
- You can choose a machine-type that has a disk equal or larger
than the original virtual machine type.

Usage:
  ./mgc virtual-machine-xaas xaas-snapshots create-id [id] [flags]

Examples:
  ./mgc virtual-machine-xaas xaas-snapshots create-id --machine-type.id="9ec75090-2872-4f51-8111-53d05d96d2c6" --machine-type.name="some_resource_name" --network.associate-public-ip=true --network.nic.id="9ec75090-2872-4f51-8111-53d05d96d2c6" --network.nic.security-groups='[{"id":"9ec75090-2872-4f51-8111-53d05d96d2c6"}]' --network.vpc.id="9ec75090-2872-4f51-8111-53d05d96d2c6" --network.vpc.name="some_resource_name"
